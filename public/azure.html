Guide: Hosting a Website on a Dedicated Virtual Machine on Microsoft Azure

Prerequisites

  1. An active Microsoft Azure account.
  2. Basic knowledge of the tech stack you choose (Node.js, Python, Golang, or Rust).
  3. A code editor for development (e.g., Visual Studio Code).

Step 1: Create a Virtual Machine on Microsoft Azure

  1. Sign in to the Azure Portal: https://portal.azure.com.
  2. Click on "Create a resource" > "Compute" > "Virtual Machine."
  3. Choose the appropriate options:
    • Subscription: Select your Azure subscription.
    • Resource Group: Create a new or select an existing resource group.
    • Virtual Machine Name: Choose a unique name for your VM.
    • Region: Select the region closest to your target audience.
    • Image: Choose an image based on your preferred tech stack (e.g., Ubuntu for Node.js and Python, etc.).
    • Size: Select an appropriate VM size based on your website's needs.
    • Authentication Type: Choose SSH public key or password for authentication.
    • Username/Password or SSH public key: Provide the necessary credentials.
    • Public Inbound Ports: Allow HTTP (80) and HTTPS (443) traffic.

Step 2: Connect to your Virtual Machine

  1. In the Azure Portal, go to your Virtual Machine.
  2. Click on "Connect" and follow the instructions to connect via SSH for Linux or RDP for Windows.

Step 3: Install Tech Stack Dependencies

For Node.js:


# Update package lists
sudo apt update

# Install Node.js and npm
sudo apt install -y nodejs npm

For Python:


# Update package lists
sudo apt update

# Install Python and pip
sudo apt install -y python3 python3-pip

For Golang:


# Download and install Go
wget https://golang.org/dl/go1.XX.X.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.XX.X.linux-amd64.tar.gz
rm go1.XX.X.linux-amd64.tar.gz

# Add Go binaries to PATH (add this to your ~/.profile or ~/.bashrc)
export PATH=$PATH:/usr/local/go/bin

For Rust:


# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Add Rust binaries to PATH (this might be added automatically by the installation)
source $HOME/.cargo/env

Step 4: Deploy your Website

  1. Upload your website files to the Virtual Machine. You can use FTP, SCP, or any other method you prefer.
  2. Place your website files in the appropriate directory based on the tech stack:
    • Node.js: Use a folder like /var/www/nodejs.
    • Python: Use a folder like /var/www/python.
    • Golang: Use a folder like /var/www/golang.
    • Rust: Use a folder like /var/www/rust.

Step 5: Run your Website

  1. Navigate to your website directory.
  2. Run the web server using the respective commands for each tech stack:

For Node.js:


# Install dependencies
npm install

# Start the server
node app.js

For Python:


# Install dependencies (if you have a requirements.txt)
pip install -r requirements.txt

# Start the server
python app.py

For Golang:


# Build and run the Go application
go run main.go

For Rust:


# Build and run the Rust application
cargo run

Step 6: Access Your Website

Your website should now be running on the dedicated virtual machine on Microsoft Azure. Access it using the public IP address or domain name associated with your VM. For example:

  • http://your_vm_public_ip/
  • http://your_domain_name/

Remember to set up proper DNS records and domain configurations for a custom domain.

Congratulations! You have successfully hosted a website using Node.js, Python, Golang, or Rust on a dedicated virtual machine on Microsoft Azure. Feel free to explore more advanced configurations and optimizations based on your website's needs. Happy coding!
